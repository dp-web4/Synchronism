## The Convergent Signal

Two independent expert personas (graduate physicist, active researcher) converged on the same
question from different attack angles on 2026-06-11:

- **Pass 3, Q3**: "What observable *does* correspond to C, operationally? Every proposed test
  I found either measures phase coherence (which C isn't) or fits C's parameters to the data
  being predicted."
- **Pass 4, Q5**: "Is there a protocol — even a sketch — for computing C from any measurable,
  in any domain? Every falsifiability claim on the site ultimately bottlenecks on this absence."

Both flagged the same concrete symptom (HIGH severity, both passes): the consciousness claim's
kill criterion is "EEG phase coherence during anesthesia" — but the framework itself insists,
correctly and repeatedly, that C is density-driven collective ordering, **not** phase coherence
(BCS, maximally phase-coherent, sits at C ≈ 0). The proposed falsification test measures a
variable the theory explicitly says C is not. And the consciousness-demo page already concedes
the deeper problem: "no calibration procedure exists to map actual EEG, fMRI, or IIT-Φ
measurements to the C-axis." One page admits the claim is unrunnable; another offers a kill
criterion. As stated, Claim 2 is unfalsifiable — wrong observable AND no map.

## Why This Is a Research Gap, Not a Site Gap

Combined with `ncorr_ladder_never_anchored.md`, the framework's central quantity is now
**doubly unanchored**:

| Side | Quantity | Status |
|------|----------|--------|
| Input | N_corr (→ γ = 2/√N_corr) | Asserted, never independently counted, on all 17 scale rungs (2026-06-08) |
| Output | C(ρ) | No operational protocol maps any measurable, in any domain, to C |

The only domain where C touches data at all is galaxy rotation — and there only through
g_eff = g_N/C(ρ), where C's parameters (ρ_crit, A) are **fit to** the rotation data being
predicted, not measured independently. Everywhere else (consciousness, chemistry, quantum
decoherence, superconductivity), C is computed from asserted inputs and compared to nothing.

The contrast that sharpens this: the anesthesia literature has an empirically calibrated
threshold candidate — PCI* = 0.31 (perturbational complexity index, Casali et al. 2013,
Sci Transl Med), validated across wake/sleep/anesthesia/disorders-of-consciousness. That is
what an operationalized threshold looks like: a measurement procedure, a number, and a
validation cohort. The framework has nothing analogous for C in any domain, and the
C-to-anything map is not even sketched.

## Decisive Question

**Does a candidate operational protocol for measuring C — independent of the data C is used
to predict — exist or follow from the framework in ANY domain?**

Three possible resolutions, all of which advance the research:

1. **A protocol is derivable** (e.g., C from correlation functions of a measured density field,
   with N_corr counted by an explicit clustering criterion). Then derive it, apply it to one
   system, and the framework gains its first operationally grounded quantity. The
   `ncorr-from-hamiltonian` constraint applies: at the galaxy rung the protocol must return
   N_corr ≈ 17 (the back-implied value from the fitted γ ≈ 0.49), contradicting the
   independent-stars premise — so any honest protocol likely *refutes* rather than rescues.
2. **No protocol is derivable** (C is a fitting intermediary, not an observable). Then the
   honest statement is category-level: claims keyed on C-thresholds (consciousness C ≈ 0.50
   foremost) are **unfalsifiable pending a calibration protocol**, which is a different and
   stronger statement than "untested," and the badges should say so.
3. **A protocol exists but only in the galaxy domain** (C inferred from rotation residuals).
   Then C is domain-local and the multi-scale "one equation" framing loses its last
   operational support.

## Why It Matters

Every falsifiability statement on the site outside galaxy rotation routes through C. If C has
no measurement protocol, the test catalog's tier structure overstates the framework's exposure
to data by an order of magnitude — most "untested" badges are actually "unrunnable as stated."
Distinguishing refuted / untested / **unrunnable** is exactly the refuted-vs-untested
discipline the project already committed to, applied one level deeper.
