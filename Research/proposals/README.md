# Research Proposals

Frame-level challenges and research direction proposals. Not findings within the current frame — challenges TO the frame.

## What belongs here

A **proposal** says "the research direction should change." A **finding** says "X is true within the current frame." Both are valuable. Findings go in `Research/` or `explorer/findings/`. Proposals go here.

## Format

```markdown
# Proposal: [one-line title]

**Author**: [session ID or human]
**Date**: YYYY-MM-DD
**Type**: direction-change | priority-shift | new-arc | closure | meta

## The Argument

[Why the current frame/direction/priority is wrong or insufficient]

## What I Propose Instead

[Specific alternative direction with first steps]

## What This Would Change

[Predictions, test targets, or resource allocation that would shift]

## Risk If Wrong

[What do we lose if this proposal is misguided?]
```

## Review

Proposals are reviewed by the human (dp) or the maintainer track. Accepted proposals update the SESSION_PRIMER. Rejected proposals stay here with a note explaining why — a rejected proposal is still a documented perspective.
